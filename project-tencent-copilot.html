媒资数据库 Copilot 设计迭代

从 LUI 对话框到 GUI 融合型助手

针对“动图素材生成 Agent”在落地过程中遇到的使用门槛与上下文割裂问题，进行系统性交互重构。将 Copilot 升级为页面级智能助手 v2.0，实现“所见即所得”的工作流融合。

Copilot Interaction Design

迭代背景：Agent 落地后的“水土不服”

“动图素材生成 Agent”初版上线后，虽然功能强大，但采用了传统的 LUI (Language User Interface) 纯文本对话模式。在实际测试中，我们发现了两个阻碍用户使用的关键问题：

1. 使用门槛高 (Prompt 依赖)

用户必须精确输入指令，例如“请把素材ID为xxx的图转动图”。非技术背景的运营人员很难一次掌握 Prompt 技巧，导致 Agent 需要多次追问澄清问题。

2. 上下文割裂 (Context Loss)

Agent 窗口在右侧，素材列表在左侧。用户在对话时无法直观看到自己操作的是哪一张图，经常需要“复制 ID -> 搜索确认 -> 再对话”，交互极其割裂。

解决方案：升级为“页面级智能助手”

交互演进：从 LUI 到 GUI 融合

在项目初期（Version 1.0），我们采用了传统的 LUI (Chatbot) 模式，用户需精确输入 Prompt（如任务类型、素材ID）。这导致了“使用门槛高”和“上下文割裂”的问题。为此，我们在 Version 2.0 进行了系统性重构。

关键特性设计

右侧悬浮气泡交互

用户可直接点击页面上的专辑，一键发起动图生产，无需输入复杂 Prompt。

主动识别与批量补齐

Agent 自动识别当前专辑下缺失动图的素材，并一键触发批量补齐，大幅减少人工操作。

表格内悬停调用 (Hover Actions)

用户在素材列表中 Hover 时，右键直接调出 Agent 进行生产，实现真正的“所见即所得”。

复盘与反思：从“0 采纳”到破局

问题：上线初期采纳率极低

上线初期发现，虽然功能可用，但最终采纳率甚至接近 0。复盘发现主要原因：

  • 用户心智惯性：更习惯沿用原有操作方式。
  • 功能边界有限：无法满足定制化需求。
  • 场景刚需度不足：部分任务量不高，非高频痛点。

关键教训 (Lessons Learned)

前期验证至关重要

需通过访谈和小规模测试验证真实痛点，将资源投入到高频刚需场景。

渐进式引导

上线后需设计引导机制，降低用户从旧习惯切换的心理成本。

核心价值：让 AI 融入业务流

通过这次迭代，我们让 Agent 从“能完成任务”走向“融入工作流”。
它不再是一个需要专门去“聊”的机器人，而是一个随手可用的、感知上下文的页面级 Copilot。这为后续其他 AI 能力（如自动打标、内容检索）的接入建立了标准的交互框架。
